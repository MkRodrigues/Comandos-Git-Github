Classificação Comando Função Observações Gerais Exemplo
Classe GLOBAL
Comandos Gerais git config --global -l Lista todas as configurações inseridas no Git O sufixo -l vem de list/lista --
-- git config --global core.editor {nome do novo editor} Configura um novo editor de texto para o Git O editor de texto pode ser usado para definir mensagens de commits e informações gerais, bem como editar conflitos git config --global core.editor code (configura o app Visual Studio Code como editor principal)
Classe GIT LOG
Git Log git log Retorna ao usuário um amontoado de infromações sobre commits, branchs e etc. As informações de commits/branchs retornadas podem conter (hash do commit/branch, autor do commit/branch, data, e estado onde se encontra o HEAD, e etc. --
Graph git log --graph Mostra logs detalhadas de forma completa, (detalhamento da árvore hierárquica de branchs e commits) -- --
Graph Oneline git log --graph --oneline Mostra logs detalhadas de forma simplificada "uma linha", (detalhamento da árvore hierárquica de branchs e commits) -- --
Q (Quit/Sair) q O comando "q" sai do detalhamento de logs -- --
Classe GIT ALIAS
Git Alias git config --global alias.{atalho para o comando} {nome do comando desejado} Cria atalhos dos comandos do git -- git config --global alias.s status
Git Alias List git config --get-regexp alias Retorna uma lista com os Alias configurados -- --
Classe GIT STASH
Git Stash git stash Salva Rascunhos dos arquivos para uso posterior Após o arquivo ser adicionado via git add, o comando Git Stash salva o arquivo ou as mudanças efetuadas naquele momento em uma área chamada WIP (Working in Progress), tirando o(s) mesmo(s) do Working Tree, salvando-o como rascunho. --
Stash List git stash list Lista os Stashs salvos -- --
Stash Apply git stash apply Aplica as mudanças salvas de volta ao Working Tree, possibilitando continuedade da edição dos arquivos -- --
Stash Clear git stash clear Limpa todos os Stash's salvos Exclui os arquivos não permitindo sua recuperação --
Classe GIT ISSUES (FEATURE DO GITHUB)
Git Issue git commit -m "Nome do commit #{numero da issue a ser referenciada}" Cria um "Problema" a ser resolvido no Github A Issue deve ser criada no Github, porém pode ser referenciada no commit do repositório local através do comando ao lado git commit -m "Primeiro commit #1"
Issue Closes git commit -m "Informação do commit closes #{número da Issue no Github}" Fecha uma Issue aberta no Github A Issue será fechada quando a branch for mergida com a master, ou se não houver branch, quando o push for realizado Para fechar uma issue referencie a mesma no commit colocando a síntaxe closes antes de #(hashtag/serquilha) - git commit -m "Informação do commit closes #1"
Issue Fixes git commit -m "Informação do commit fixes #{número da Issue no Github}" Fecha uma Issue aberta no github, indicando que a mesma faz parte de um bug que está sendo resolvido A Issue será fechada quando a branch for mergida com a master, ou se não houver branch, quando o push for realizado Para fechar uma issue referencie a mesma no commit colocando a síntaxe fixes antes de #(hashtag/serquilha) - git commit -m "Informação do commit fixes #1"
